# Contributing to Glanceable

Glanceable is intentionally minimal.

Before contributing, understand that this is not a typical social or productivity app.  
Most features are **excluded by design**, not missing.

---

## Philosophy

Glanceable follows a few strict constraints:

- No gamification  
- No notifications  
- No feeds or engagement loops  
- No scoring, ranking, or streaks  
- No pressure to respond  

The goal is simple:

> The smallest possible signal that helps people decide whether to connect.

If a feature increases noise, pressure, or engagement mechanics — it will not be accepted.

---

## What to work on

Good contributions:

- Reduce friction
- Improve clarity
- Make interactions faster or more obvious
- Fix bugs or edge cases
- Improve visual calm / readability

Avoid:

- Adding new systems or layers
- Adding “social” features
- Increasing complexity

---

## Code expectations

- Keep changes small and focused
- Prefer simple over clever
- Match existing patterns in the codebase
- Don’t introduce new abstractions unless clearly needed

---

## Before submitting

- Run `npm run build`
- Make sure the app works locally
- Avoid unrelated refactors

---

## Tone

Glanceable is designed to feel:

- calm
- lightweight
- non-intrusive

Code changes should support that feeling.

---

Thanks for contributing.
